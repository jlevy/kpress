# KPress 0.1.0

KPress `0.1.0` is the first standalone alpha for developers testing Markdown document
rendering and static publishing in outside projects.

Supported paths:

- Install the tagged Python package on Python 3.12–3.14.
- Build a multi-page static site through `kpress build` or `build_site`.
- Render standalone pages and host-wrapped fragments through the typed Python API.
- Customize public CSS tokens, page chrome slots, widgets, behaviors, and documented
  build-pipeline stages.
- Use `linked`, `hashed`, or host-served KPress package assets.

Important alpha limits:

- Public seams are provisional and may change by hard cut before a stable release.
- `hashed` fingerprints KPress-owned assets but does not fetch or verify external or
  document-local assets.
- Self-contained single-file and verified offline output are not implemented.
- The live standalone shell is `page.html.jinja`; fragment and component markup still
  comes from the Python renderer pending a later template migration.
- Sanitizer diagnostics identify removed tags and attributes, but sanitized output is
  not proof that a document was authored by a trusted party.

Report integration defects and contract gaps at
[github.com/jlevy/kpress/issues](https://github.com/jlevy/kpress/issues).
Include the KPress version, Python version, config, smallest reproducer, and relevant
manifest diagnostics.
