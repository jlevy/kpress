# KPress 0.2.1

KPress `0.2.1` is a compatible maintenance release for the `0.2` host-rendering and
asset-contract line.
It fixes conditional tooltip asset selection and raises the project’s build, validation,
and release floor.

## Rendering Fix

- The `auto` asset policy now includes the tooltip runtime whenever sanitized output
  contains a same-document link such as `href="#section"`.
- Fragment hosts therefore receive the runtime needed for heading-link previews even
  when a page has no footnote reference.
- Regression coverage pins the asset-manifest behavior.

## Tooling and Release Quality

- CI and local verification now share Makefile entry points and exact current Node, npm,
  uv, lint, type-checking, and test-tool floors.
- The release gate audits frozen Python and npm dependency graphs, inspects wheel and
  source-distribution contents, and exercises an isolated install.
- Public-hygiene checks now cover source, tests, and documentation consistently.

## Compatibility

This release does not change the typed asset-manifest schema, renderer API, or public
fragment hooks introduced in `0.2.0`. Hosts pinned to `0.2.0` can upgrade directly.

## Verification

The complete Python and JavaScript quality gates, dependency audits, distribution
inspection, clean-room install, CLI smoke tests, and package capability tests run before
publication. The exact PyPI artifact is verified separately after trusted publication.
