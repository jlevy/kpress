# KPress 0.2.0

KPress `0.2.0` makes the fragment asset boundary complete and typed for embedding
applications. It remains an alpha and uses the existing hard-cut compatibility policy.

## Host Rendering

- `RenderedDocument.assets` and `RenderedPage.assets` are now an `AssetManifest` rather
  than loose CSS and JavaScript path buckets.
- Each manifest declares the complete package closure, including stylesheets, fonts,
  transitive ES modules, conditional math files, browser loading modes, content hashes,
  output paths, public URLs, entry-point roles, and required import-map mappings.
- `materialize_package_assets` writes and verifies exactly the package files in a
  resolved manifest. The wrapped-site example now uses this public API instead of parsing
  CSS URLs and JavaScript imports.
- `RenderOptions.asset_policy` replaces `include_assets`. `auto` selects styles and only
  feature-required modules, `none` returns an empty manifest, and `all` returns the full
  reader and math closure.
- `PUBLIC_FRAGMENT_CSS_CLASSES` and `PUBLIC_FRAGMENT_CSS_VARIABLES` pin the minimum
  stable structure and theme tokens an embedding shell needs for prose, tables, figures,
  spacing, and print.

## Migration

Hosts must serve or materialize every manifest asset but emit browser tags only for
entries whose `entry_point` is true.
The `loading` field distinguishes `stylesheet`, `module`, and `classic` tags; `resource`
and dependency-only entries receive no direct tag.
Install a non-empty `import_map` before module entry points.

Replace code that indexes `result.assets["css"]` or `result.assets["js"]` with
`result.assets.browser_entry_points()`. Replace `RenderOptions(include_assets=False)`
with `RenderOptions(asset_policy="none")`.

## Verification

The package suite covers prose and rich render closure, hashed module resolution, font
and math dependencies, exact materialization, symlink/path rejection, the public host
fragment fixture, static-site minimal asset output, and the wrapped-site example.
The full Python and JavaScript quality gates run before publication.
