# KPress 0.2.3

KPress `0.2.3` is a compatible maintenance release for the `0.2` host-rendering and
asset-contract line.
It makes in-document navigation history-aware, normalizes TOC nesting into a proper
tree, scopes numeric table alignment to whole columns, and keeps the document clear of
the floating TOC toggle on narrow screens.

## History-Aware Section Navigation

- A new `history` behavior (`js/history.js`, export `initKpressHistory`) records the
  reading pane’s scroll offset in the session-history entry, so Back and Forward restore
  the reader’s position after section-link jumps; the scroll pane is invisible to native
  browser scroll restoration.
- TOC links and plain section links now perform native hash navigation: real history
  entries, shareable URLs, and a smooth in-pane glide that honors
  `prefers-reduced-motion`. Footnote references keep their popover owner, and modified
  clicks, `download` anchors, and non-`_self` targets stay fully native.
- The behavior ships automatically with any page whose asset manifest includes the TOC
  or same-document-link runtimes, never overwrites host-owned `history.state`, and
  returns a disposer for embedding hosts that swap documents per view.

## TOC Structure and Clearance

- TOC nesting is now structural tree depth: each entry sits one level under its nearest
  preceding shallower heading.
  Documents that skip heading levels no longer over-indent, and a heading with no
  enclosing ancestor sits at the top level.
  Rendered heading tags are unchanged; the design reference documents this as contract
  behavior.
- The document reserves an inline gutter for the floating TOC toggle through the new
  `--kpress-toc-toggle-clearance` variable (public and fragment-safe), so prose never
  slides under the control at narrow and phone widths.
  Hosts whose pane inset already provides the gutter can shrink the reservation.

## Rendering Fixes

- Numeric table alignment is decided per column instead of per cell: a column aligns
  numerically only when at least one non-empty body cell matches the numeric pattern and
  none mismatches, header cells follow their column, tables using row or column spans
  are left unmarked, and the typographic minus (U+2212) is accepted.
  Server output and the client runtime converge on the same decision.
- Printed pages align the text column with the `@page` margin boxes instead of keeping
  the screen-side inline gutters.
- TOC and footnote-reference transitions are scoped to their hover properties, so a
  late-applied stylesheet snaps into place instead of visibly animating.

## Tooling and Release Quality

- The repository gates consolidate into a single supply-chain checker covering npm and
  uv policy, lock integrity, workflow pinning, and trusted-publishing controls, with the
  uv floor raised to `0.11.26`.
- Real-browser Playwright coverage now guards toggle clearance and history traversal
  alongside the browserless DOM suites.

## Compatibility

The typed asset-manifest schema, renderer API, publisher output, and fragment hooks from
the `0.2` line are unchanged.
The public surface grows additively: the `history` behavior, its `js/history.js` export,
and the `--kpress-toc-toggle-clearance` variable.
Rendered output changes only where the old output was wrong: skipped heading levels
produce shallower TOC indentation, and mixed-content table columns lose per-cell numeric
alignment. Hosts pinned to `0.2.0` through `0.2.2` can upgrade directly.

## Verification

The complete Python and JavaScript quality gates, dependency audits, distribution
inspection, clean-room install, CLI smoke tests, package capability tests, and
real-browser navigation and clearance checks run before publication.
The exact PyPI artifact is verified separately after trusted publication.
